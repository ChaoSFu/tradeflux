# TradeFlux 评分体系说明

> 版本：v2.0  
> 更新：2026-05-27  
> 计算位置：`backend/app/services/screening_service.py → compute_window_stats()`

---

## 一、总体设计原则

TradeFlux 对强势股池中的每只股票计算三个独立维度的评分，用于辅助短线操作决策。

```
龙头分  →  我现在是不是这波行情的领头羊？
情绪分  →  市场对我有多热情？量能在放大还是缩量？
风险分  →  当前持有有多危险？
```

三个分数**设计为相互正交**（信息不重叠），避免重复计量同一信号。
每次每日更新（`daily_update.py`）后自动重算并写入数据库。

---

## 二、情绪分（Emotion Score）

**定义**：衡量市场参与度与量能热度，反映当前有多少资金关注这只票。

**范围**：0 – 100

**公式**：

```python
emotion = min(100.0,
    lup_60 × 1.2            # 近60日涨停天数（历史活跃度）
    + board_60 × 5.0        # 近60日最高连板数（历史极值）
    + 12.0  (若今日涨停)    # 当日热度加成
    + turn_expansion        # ★ 量能扩张信号（0–15）
    + turnover × 0.8        # 今日换手率（当日有效时）
)
```

其中 **量能扩张信号** 为核心改进因子：

```python
# 近5日均换手 vs 近20日均换手（排除今日）
turn_ratio     = avg(turnover[-6:-1]) / avg(turnover[-21:-1])
turn_expansion = min(15.0, max(0.0, (turn_ratio - 0.5) × 10.0))
```

| turn_ratio | 含义         | turn_expansion |
|-----------|--------------|---------------|
| 0.5x      | 量能大幅萎缩  | 0 分           |
| 1.0x      | 量能平稳      | 5 分           |
| 1.5x      | 量能明显放大  | 10 分          |
| 2.0x+     | 量能爆发      | 15 分          |

**设计意图**：区分"历史表现好"与"当前市场热情高"。一只票过去有很多涨停但现在缩量横盘，情绪分会低于一只近期量能持续放大的票。

## 三、风险分（Risk Score）

**定义**：衡量下行风险烈度，越高越危险。

**范围**：0 – 100

**公式**：

```python
risk = min(100.0,
    broken_3d  × 28.0               # 近3日炸板（高危）
    + broken_7d × 12.0              # 3–10日炸板（中危）
    + max(0, board_60 - 4) × 8.0   # 高板位分歧风险
    + 15.0  (若今日跌停)
    + min(30.0, declines × 8.0)     # ★ 连续下跌天数
)
```

**炸板时间权重**：

| 炸板时间 | 每次权重 | 说明 |
|--------|---------|------|
| 近3日  | ×28     | 主力刚刚出货信号，极度危险 |
| 3–10日 | ×12     | 风险已部分消化 |

**连续下跌因子**（`consecutive_declines`）：

| 连跌天数 | 风险加分 | 建议 |
|---------|---------|------|
| 1–2天   | 8–16   | 观望 |
| 3天     | 24     | 触发 weakening 阶段 |
| 4天+    | 30（上限）| 大概率出局 |

**已修复的错误**：v1.0 中存在 `max(0, 60-lup_60) × 0.3`（"活跃度不足"）因子，
该因子逻辑相反——刚入池、刚开始加速的新票会被误判为高风险。已在 v2.0 中移除。

## 四、龙头分（Leader Score）

**定义**：综合衡量当前板块领涨地位，越高越强。

**范围**：0 – 100

**公式**：

```python
leader = min(100.0,
    min(30, board_now × 11)            # 当前连板数（≤30）
    + (lup_10 / 7.0) × 30             # 近10日涨停密度（≤30）
    + max(0, (emotion-20)/60) × 20    # 情绪归一化（≤20）
    + max(0, (board_60-1)/7) × 12     # 历史板高（≤12）
    + max(0, (lup_60-3)/18) × 8       # 60日涨停密度（≤8）
    + 5.0   (若今日涨停)
    + min(5, turnover × 0.5)          # 今日换手（≤5）
    + 12.0  (若 is_sector_leader)      # ★ 板块龙头加成
    - 12.0  (若今日炸板)
)
```

**板块龙头加成**：`StockSectorRelation.is_leader` 在 `sync_boards.py` 同步概念板块时设置，
标记该股是否为该板块的成交额/涨幅最强票。龙头股在板块联动行情中成功率显著高于跟风股。

**多因子叠加设计**：需要同时具备"当前连板 + 近期活跃 + 情绪强"才能达到高分，
单一历史指标好但当前不活跃的票会自然落入中低分区间。

**典型分值参考**：

| 场景 | 典型得分 |
|-----|---------|
| 板块龙头 + 3连板 + 今日涨停 | 85–100 |
| 活跃股 + 近期多板 + 板块龙头 | 60–80  |
| 历史强 + 当前震荡横盘 | 30–50  |
| 仅刚满足入池条件 | 10–25  |

## 五、操作参考矩阵

三个分数组合对应不同操作意图：

| 龙头分 | 情绪分 | 风险分 | 建议操作 |
|-------|-------|-------|---------|
| 高 (>55) | 高 (>50) | 低 (<35) | **积极追涨** — 接板或次日竞价关注 |
| 高 (>55) | 低 (<40) | 低 (<35) | **潜伏等待** — 等量能起来再介入 |
| 高 (>55) | 高 (>50) | 高 (>60) | **谨慎持有** — 高位高风险，轻仓或止盈 |
| 任意    | 下降趋势 | 高 (>60) | **出局信号** — 主力出货迹象明显 |
| 低 (<30) | 低       | 上升趋势 | **移出观察** — 不再符合强势定义 |

> ⚠️ 以上仅为辅助参考，不构成投资建议。最终决策需结合盘面、板块联动、大盘环境判断。

## 六、强势股池入池条件

入池条件为 OR 逻辑，满足任意一条即可进入强势池：

```
1. 60日最高连板数 > min_board_count_60d（默认 3，即≥4板）
2. 60日涨停天数   > min_limit_up_days_60d（默认 9，即≥10天）
3. 10日涨停天数   > min_limit_up_days_10d（默认 4，即≥5天）
4. 20日涨幅 排名前 top_pct_rank_20d%（默认前10%）
```

排除条件（AND）：
- `exclude_st = True`：排除 ST 股
- `exclude_new_stock = True`：排除上市不足 N 个月的新股

筛选条件存储在 `ScreeningCriteria` 表中，可在管理界面动态调整。

## 七、阶段分类（Phase）

每只强势股被分配一个阶段标签，驱动前端 Tab 分组：

| 阶段 | 条件 | 说明 |
|-----|-----|------|
| `normal` → 震荡龙头 | 在均线之上，且不满足下面两个条件 | 活跃期，主要关注对象 |
| `weakening` → 走弱龙头 | 在均线之上，但连续下跌 ≥3 天 | 观察，等待企稳信号 |
| `broken` → 破位龙头 | 收盘价 < 60日均线 | 高风险，考虑移出 |
| `limit_up_today` → 涨停龙头 | 今日快照 `is_limit_up = True` | 最高优先级展示 |

## 八、后续优化路径

### P1 — 高优先级（数据已有，改动小）

**[ ] 相对强度因子**  
在同板块内比较，真正的龙头是"比板块内其他股涨更多的那只"。  
- 实现：用 `StockSectorRelation` + 快照数据，计算每只股10日涨幅 vs 板块平均  
- 加入龙头分公式：`relative_strength = (stock_10d_pct - sector_avg_10d_pct) / sector_std`  
- 影响文件：`screening_service.py`，需要在 `daily_update.py` 中传入板块均值

**[ ] 炸板后恢复信号**  
炸板后重新封板（当日或次日）是分歧出清的信号，应降低风险分而非累计惩罚。  
- 实现：检查炸板次日是否重新涨停，若是则将该炸板权重减半  
- 影响文件：`screening_service.py`

### P2 — 中优先级（需要额外数据或较大重构）

**[ ] 连板质量：一字板比例**  
一字板（开盘即封死）反映买方绝对力量，普通涨停风险更高。  
- 需要 `open_price` 字段（当前 KLineBar 已有）  
- 实现：`one_word_ratio = count(open == high == limit_price) / board_count`  
- 加入龙头分/情绪分公式，加入风险分作为反向因子

**[ ] 板块热度加权**  
个股评分应考虑板块当前阶段：一只60分的股票在热点板块里比在冷板块里更值得关注。  
- 实现：`sector_heat_multiplier = f(sector.phase, sector.emotion_score)`  
- 综合得分 = 龙头分 × sector_heat_multiplier  
- 影响文件：`strong_stock_service.py`（在 enrich 阶段注入）

**[ ] 操作信号（Action Signal）**  
将三分合并为一个离散的操作建议信号，直接指导决策。  
- 值：`积极 / 观望 / 减仓 / 出局`  
- 实现：规则引擎 + 可配置阈值  
- 展示：强势池列表新增一列，高亮显示

**[ ] 时间衰减权重**  
60日内越近的涨停应权重越高，当前用等权计算。  
- 实现：`weighted_lup_60 = Σ (is_limit_up[i] × exp(-λ × days_ago[i]))`  
- 需要对近期活跃票提分，对仅历史数据好的票降分

### P3 — 低优先级（需要新数据源）

**[ ] 主力资金流向**  
A 股中主力净流入是最强情绪信号之一，当前无此数据。  
- 数据源：东方财富资金流向接口（已有爬虫基础设施，可扩展）  
- 加入情绪分，权重建议 10–15 分

**[ ] 分时开板/回封次数**  
衡量连板强度的精细指标：开板后是否快速回封，回封速度越快越强。  
- 需要日内分时数据，当前只有日 K 线

**[ ] 大盘环境联动调整**  
牛市/熊市背景下，同样的个股指标含义不同。  
- 实现：根据 `DailyReview.market_phase` 动态调整入池阈值和评分权重  
- 暖场/狂热时适当降低门槛扩大池子，熊市时提高门槛保留真正强势股
